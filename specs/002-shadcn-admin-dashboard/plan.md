## Summary

Build a standalone web admin dashboard using React and shadcn/ui components that connects to the existing NestJS backend. The dashboard displays platform metrics from the `/admin/dashboard` endpoint including active students, ongoing exams, recent activations, and security flags. Admins can view data in real-time and mark security flags as reviewed.

## Technical Context

**Language/Version**: TypeScript 5.x, React 18.x  
**Primary Dependencies**: shadcn/ui, React, Vite, Lucide React (icons)  
**Storage**: API-only (connects to existing NestJS backend with MongoDB)  
**Testing**: Vitest, React Testing Library

**Performance Goals**: < 3s initial load, 60s auto-refresh

## Phase 0: Research Findings

### shadcn/ui Integration

- shadcn/ui is a component library that works with React + Vite or Next.js
- Installation requires initializing a new React project with Vite or using existing Next.js
- Components are copied into the project (not a traditional npm package)
- Uses Tailwind CSS for styling
- Provides well-documented patterns for data tables, cards, forms

### Best Practices for Admin Dashboards

- Use shadcn Card component for metric displays
- Use Table component for data lists (activations, security flags)
- Implement auto-refresh with useEffect + setInterval
- Handle loading/error/empty states with shadcn UI components
- Use React Query or SWR for data fetching and caching
